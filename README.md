## Commit Message

- feat : 기능 추가

- fix : 기능 수정

- refactor : 리팩토링

- style : ui 관련 전체

- chore : 세팅 관련

## Feature 브랜치 이름 형식

    - `이슈라벨/기능명-이슈번호`
        - 예: `feat/add-calendar-23`

## Issue & PR

### Issue 생성 형식

- `<타입>:<내용>` 형식으로 발행
- 라벨 및 담당자 지정 필수

### PR 작성 규칙

- 제목 형식: `<타입>:<내용>` (이슈 제목과 통일)
